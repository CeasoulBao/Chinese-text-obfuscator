# 字隙 · 文本混淆器

一个完全在浏览器本地运行的可组合文字混淆与图片排版工具。

## 功能

- 自定义词典替换。
- 同音字与形近字替换。
- 字符分组乱序。
- 随机符号穿插。
- 轻度、中度、重度及自定义强度。
- 可调整模块顺序，并使用随机种子复现结果。
- 将结果排版为 PNG 图片。
- 添加横向穿行线、纵向穿列线、噪点和字符抖动。
- 导出一张长图，或按指定高度自动切成多张图片。

## 隐私

应用没有后端、上传接口、分析脚本或原文持久化功能。文字处理和图片生成都发生在当前浏览器标签页内。点击“清空”或关闭页面即可丢弃内存中的内容。

## 重要边界

本项目提供的是**文本混淆**，不是加密，也不提供匿名性。

固定替换、插入符号、字符乱序和图片叠线都可能被现代文本分类、图像预处理、OCR 或人工复核还原。使用本工具不能保证绕过内容审核、避免账号限制或隐藏平台可见的发送者、接收者、时间、设备及社交关系信息。请在导出或发送前检查结果是否仍可被目标读者理解。

## 本地运行

需要 Node.js 20.19+ 或 22.12+。

```bash
npm install
npm run dev
```

浏览器打开终端显示的本地地址，通常是 `http://localhost:5173`。

## 测试与构建

```bash
npm run test:run
npm run build
npm run preview
```

生产文件位于 `dist/`。

## 部署到 GitHub Pages

1. 在 GitHub 创建仓库并推送本项目。
2. 打开仓库的 **Settings → Pages**。
3. 将 Source 设为 **GitHub Actions**。
4. 推送到 `main` 或 `master` 分支。
5. `.github/workflows/deploy.yml` 会自动测试、构建并发布 `dist/`。

Vite 使用相对资源路径，因此仓库项目页与自定义域名都可以直接工作。

## 项目结构

```text
src/
  components/       React 界面组件
  image/            排版、Canvas 渲染与导出
  obfuscation/      随机源、模块注册表和文字转换
  state/            应用状态与预设
  types/            共享类型
```
